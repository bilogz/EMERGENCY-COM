# Documentation Files (Markdown)

This directory contains all Markdown documentation files organized by category.

## Directory Structure

```
docs-md/
├── admin/          - Admin panel documentation
├── users/          - User portal documentation
├── deployment/     - Deployment guides and checklists
├── guides/         - Feature guides and how-tos
└── README.md       - This file
```

## Organization

### Admin Documentation (`admin/`)
- Admin panel setup guides
- Admin feature documentation
- Admin system guides

### Users Documentation (`users/`)
- User portal documentation
- User feature guides
- User API documentation

### Deployment (`deployment/`)
- Deployment guides
- Deployment checklists
- Server configuration guides

### Guides (`guides/`)
- Feature implementation guides
- Technical documentation
- System architecture docs

## Usage

All Markdown documentation files should be organized in the appropriate subdirectory based on their purpose and audience.

## Note

Some documentation files may still be in their original locations (like `DOCS/` folder) for backward compatibility. This folder structure is for better organization going forward.
